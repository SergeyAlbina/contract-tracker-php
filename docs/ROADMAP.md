# Roadmap: Contract Lifecycle Tracker

## Принципы
- v1: быстрый MVP + красивый UI + твёрдый фундамент (архитектура/валидация/аудит)
- v2: адаптация под реальные процессы руководителя КС + бухгалтерия/филиалы/интеграции
- место для манёвров: feature flags и политики (настройки) вместо “переписать всё”

---

## v1 (MVP) — “скелет + стиль”
### Backend
- Auth: login/password, JWT + refresh
- RBAC: ADMIN, HEAD_CS, SPECIALIST_CS
- Закупки (Procurements): CRUD + статусы
- КП (Commercial Proposals): CRUD, статусы PENDING/ACCEPTED/REJECTED, замечания
- Контракты: CRUD + статусы жизненного цикла
- Этапы (обязательные): CRUD, просрочки
- Счета/Акты/Оплаты: CRUD, статусы оплат PLANNED/IN_PROGRESS/PAID/CANCELED
- Остаток и флаги риска (expiring/overspend/overdue/missing docs)
- Документы (Storage v1): upload/download, метаданные, ссылки на Диадок/1С
- Экспорт:
  - реестр → XLSX/CSV
  - пакет контракта → ZIP (документы + метаданные + паспорт HTML)
- Уведомления:
  - Telegram: ежедневный дайджест + критические события
- Аудит: журнал изменений

### Frontend (Material You)
- AppShell (drawer/topbar), theme light/dark, responsive layout
- Dashboard: KPI cards + таблица “Тревоги”
- Страницы: Закупки, КП, Контракты, Карточка контракта (tabs), Документы
- Единые компоненты: DataTable, StatusChip, RiskBadge, KpiCard

---

## v1.1 (усиление без ломки)
- PDF паспорт контракта (рендер из HTML)
- Улучшенные фильтры/поиск (по ИНН, номеру, статусам, флагам)
- “Workload” (нагрузка): score по специалистам + автоназначение владельца
- Справочники: типы документов, статусы, причины отклонений КП

---

## v2 (под руководителя КС / расширение)
- Бухгалтерия как роль (ACCOUNTANT)
- Филиалы и матрица доступа
- 2FA (TOTP + recovery codes) для head/admin
- MinIO storage (S3) + presigned URLs + политики доступа
- Интеграции:
  - импорт/выгрузка под шаблон 1С
  - расширенный экспорт отчётов (шаблоны)
  - улучшение связей “Диадок/ЭДО”
- Процессы:
  - согласования, претензии, допсоглашения как сущности
  - задачи (tasks) + SLA
- Тонкая настройка правил (policy engine): что обязательно для 223/44

---

## v3 (масштаб/устойчивость)
- Очереди (BullMQ/Redis) для тяжёлых задач (PDF, ZIP, импорты)
- Multi-tenant (если понадобится “под другого человека” как отдельный контур)
- Полное журналирование/отчётность под регламенты учреждения
