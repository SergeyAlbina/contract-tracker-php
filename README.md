# Реестр контрактов (PHP 8.5)

Система управления жизненным циклом контрактов для 223-ФЗ / 44-ФЗ без фреймворков.

## Текущий функционал

- Аутентификация: вход/выход, ограничение попыток, аудит.
- Контракты: создание, редактирование, удаление, фильтры, карточка, финансы.
- Закупки: реестр закупок и коммерческие предложения (с выбором победителя).
- Дела (Cases): единый реестр для вкладок Задачи/Расторжения/Претензии/Заключённые/ФЗ с атрибутами, событиями, файлами и исполнителями.
- Этапы исполнения: план/факт по контракту, создание/редактирование/удаление.
- Счета и акты: два отдельных реестра в карточке контракта с CRUD.
- Платежи: статусы, привязка к контракту, контроль оплат.
- Документы: загрузка, скачивание, удаление.
- Пользователи: управление пользователями (для администратора), смена пароля.
- Экспорт: выгрузка реестра контрактов в файл.
- Безопасность: CSRF, RBAC, подготовленные SQL-запросы, защита сессий.

## Требования

- PHP `8.5+` с расширениями: `pdo_mysql`, `json`, `mbstring` (желательно).
- MySQL `8.0+` (или совместимый MariaDB).
- Веб-сервер с `document root` на папку `public/`.

## Быстрый запуск

1. Подготовить конфигурацию:
```bash
cp .env.example .env
```

2. Создать базу и пользователя:
```sql
CREATE DATABASE contract_tracker CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'contract_user'@'127.0.0.1' IDENTIFIED BY 'contract_pass';
GRANT ALL PRIVILEGES ON contract_tracker.* TO 'contract_user'@'127.0.0.1';
FLUSH PRIVILEGES;
```

3. Импортировать схему:
```bash
mysql -h127.0.0.1 -ucontract_user -pcontract_pass contract_tracker < database/schema.sql
```

4. Запустить приложение:
```bash
php -S 127.0.0.1:8000 -t public
```

5. Открыть:
- `http://127.0.0.1:8000/login`
- логин: `admin`
- пароль: `admin123`

## Импорт из реестров Excel

Подготовка очищенного датасета (из входящих/претензий/задач):

```bash
python3 scripts/prepare_cases_dataset.py \
  --incoming "/path/Реестр входящих и отписаных документов.xlsx" \
  --claims "/path/Реестр притензий 2024-2025.xlsx" \
  --tasks "/path/ЗАДАЧИ 2025.xlsx" \
  --out-dir output/spreadsheet
```

После подготовки:
- `output/spreadsheet/cases_dataset_clean.xlsx` — сводный очищенный файл для проверки;
- `output/spreadsheet/cases_import.csv` — источник для загрузки в `cases`;
- `output/spreadsheet/import_report.json` — контроль качества/объёмов.

Загрузка в систему:

```bash
# только проверка
php scripts/import_cases_csv.php output/spreadsheet/cases_import.csv --dry-run

# применение в БД
php scripts/import_cases_csv.php output/spreadsheet/cases_import.csv --apply
```

## Ключевые маршруты

- `GET /contracts` — список контрактов.
- `GET /contracts/export.csv` — выгрузка реестра.
- `GET /procurements` — список закупок.
- `GET /cases?block_type=&year=&status=&assignee=&q=` — единый реестр дел (JSON).
- `GET /cases/registry` — веб-интерфейс реестра дел.
- `GET /cases/registry/{id}` — карточка дела (веб).
- `POST /cases` — создать дело.
- `PATCH /cases/{id}` — обновить дело.
- `POST /cases/{id}/attributes` — сохранить редкие атрибуты.
- `POST /cases/{id}/events` — добавить событие.
- `POST /cases/{id}/files` — загрузить файл.
- `POST /cases/{id}/assignees` — назначить исполнителей.
- `POST /contracts/{id}/stages` — добавить этап исполнения контракта.
- `POST /contracts/{id}/invoices` — добавить счёт.
- `POST /contracts/{id}/acts` — добавить акт.
- `POST /contracts/{id}/payments` — добавить платёж.
- `POST /contracts/{id}/documents` — загрузить документ.
- `GET /audit` — журнал аудита (администратор).
- `GET /users` — управление пользователями (администратор).
- `GET /profile/password` — смена пароля текущего пользователя.

## Дорожная карта

- `v1.0` (выполнено): аутентификация, контракты, платежи, документы, базовая безопасность.
- `v1.1` (выполнено): пользователи, экспорт, усиление ролевых проверок.
- `v2.0`:
  - выполнено: закупки + КП;
  - выполнено: этапы контракта (план/факт);
  - выполнено: счета и акты;
  - выполнено: журнал аудита (интерфейс для администратора).
- `v2.1` (план): сводный экран, паспорт контракта, почтовые уведомления.
- `v3.0` (план): API, нагрузка сотрудников, хранение файлов в объектном хранилище, версионирование контрактов.

## Минимальный чек-лист для продакшна

1. Сгенерировать и установить безопасный `APP_SECRET`.
2. Установить `SESSION_SECURE=true`.
3. Установить `APP_DEBUG=false`.
4. Сразу сменить пароль пользователя `admin`.
5. Держать каталог `storage/` вне публичного доступа.
